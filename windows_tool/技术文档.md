# ESI 高水平论文统计工具技术文档

## 1. 目标

这套工具面向非计算机专业人员，目标是把原本需要手工执行的 ESI 年度统计流程，收敛成一个 Windows 桌面工具。当前版本聚焦 `2025` 年统计口径，能够自动完成以下任务：

- 读取 6 期 ESI 附表
- 去重并生成 `summary.json` 与 `papers.json`
- 生成统计工作簿 `2025年度ESI高水平论文统计结果.xlsx`
- 生成正式报告 `我校2025年度ESI高水平论文的产出情况及影响力分析.docx`
- 生成 `统计口径说明` 与 `作者统计流程详解`

工具定位是 `先能稳定跑流程`，再逐步扩展为更通用的年度统计平台。

## 2. 目录结构

项目中和 Windows 原型直接相关的文件如下：

```text
analyze_2025_esi.py
generate_2025_esi_doc.py
generate_2025_esi_process_doc.py
generate_author_flow_doc.py
generate_esi_stats_workbook.py
windows_tool/
  __init__.py
  esi_pipeline.py
  esi_gui_app.py
  requirements-windows.txt
  run_esi_tool.bat
  build_windows_exe.bat
  技术文档.md
```

职责说明：

- `analyze_2025_esi.py`
  - 核心分析脚本
  - 负责读取附表、按题名去重、归并作者、生成 `summary.json` 和 `papers.json`

- `generate_esi_stats_workbook.py`
  - 纯 Python Excel 生成器
  - 面向 Windows，不依赖 Node
  - 生成包含表12、表13、表14、去重总表、拼音待核明细的工作簿

- `generate_2025_esi_doc.py`
  - 生成正式年度报告

- `generate_2025_esi_process_doc.py`
  - 生成统计口径说明文档

- `generate_author_flow_doc.py`
  - 生成作者统计流程详解文档

- `windows_tool/esi_pipeline.py`
  - GUI 背后的流水线编排层
  - 顺序调用上面这些脚本

- `windows_tool/esi_gui_app.py`
  - Tkinter 图形界面
  - 非技术人员直接操作这个入口

- `windows_tool/run_esi_tool.bat`
  - Windows 启动脚本
  - 自动创建虚拟环境并启动 GUI

- `windows_tool/build_windows_exe.bat`
  - Windows 打包脚本
  - 用 PyInstaller 生成桌面 exe

## 3. 运行流程

### 3.1 用户视角

老师在 Windows 上使用时，只需要：

1. 双击 `run_esi_tool.bat`
2. 选择：
   - `附表文件夹`
   - `2024模板报告`
   - `输出工作目录`
3. 点击 `开始生成`
4. 等待日志结束
5. 到输出目录查看结果

### 3.2 系统视角

GUI 实际执行的是以下 5 个步骤：

1. `analyze_2025_esi.py`
2. `generate_esi_stats_workbook.py`
3. `generate_2025_esi_doc.py`
4. `generate_2025_esi_process_doc.py`
5. `generate_author_flow_doc.py`

生成目录结构如下：

```text
<工作目录>
  analysis/
    summary.json
    papers.json
  outputs/
    2025年度ESI高水平论文统计结果.xlsx
    我校2025年度ESI高水平论文的产出情况及影响力分析.docx
    2025年度ESI高水平论文统计口径与核验说明.docx
    2025年度ESI高水平论文作者统计流程详解.docx
```

## 4. 核心数据流

### 4.1 原始输入

输入是一组 Excel 附表。当前版本假设文件夹中存在以下 5 个文件模式：

- `ESI*第1期附表*.xlsx`
- `ESI*第2期附表*.xlsx`
- `ESI*第3期附表*.xlsx`
- `ESI*第4-5期附表*.xlsx`
- `ESI*第6期附表*.xlsx`

第 4、5 期共用同一工作簿，读取其中两个不同 sheet。

### 4.2 分析产物

`analyze_2025_esi.py` 生成两个核心文件：

- `summary.json`
  - 汇总统计结果
  - 包括总量、分布、院系-学科计数、院系-作者计数、拼音待核清单等

- `papers.json`
  - 去重后的论文级明细
  - 供 Excel 总表展示使用

### 4.3 输出文档

- Excel 工作簿从 `summary.json + papers.json` 生成
- 报告类 docx 只依赖 `summary.json`

## 5. 作者统计逻辑

作者统计是当前系统里最复杂的部分，逻辑分成 5 层：

### 5.1 作者字符串拆分

脚本把附表中的作者列按分号拆分成多个 token，例如：

```text
Xu,X(Xu, Xin 徐鑫); Xuan,SY(Xuan, Siyuan 宣思源)
```

每个 token 再提取：

- 中文名
- 拼音别名

### 5.2 别名映射

如果同一个 token 同时出现拼音和中文，例如：

```text
Wu,PP( Wu, Peipei 吴佩佩)
```

程序就建立：

```text
wupeipei -> 吴佩佩
```

此外还叠加少量人工确认的映射，当前写在：

```python
AUTHOR_ALIAS_MANUAL
```

### 5.3 论文级聚合

按标准化题名把同一篇论文在不同期次的记录聚合成一条年度论文对象。

聚合后会同时保留：

- `chinese_names`
- `unresolved_author_tokens`

### 5.4 表14代表作者选择

表14不是“所有作者展开表”，而是“同一篇论文在同一学院内只记1位代表作者”。

选择顺序是：

1. 先选能明确映射到该学院的中文作者
2. 再选 2024 种子表中已知归属该学院的中文作者
3. 若仍不行，则选第一个中文作者
4. 若整篇都没有中文作者，才退回拼音作者

对应代码：

- `infer_author_mapping()`
- `choose_author_for_college()`

### 5.5 拼音待核清单

当前原型把两类论文放进拼音待核清单：

1. `整篇都没有可靠中文作者`
2. `已经识别出部分中文作者，但仍有少量拼音作者待核`

其中第二类只有在人工指定的白名单标题中才会被显式列出，避免把所有外校作者都误收进去。

当前白名单在：

```python
MANUAL_PENDING_AUTHOR_OVERRIDES_RAW
```

## 6. 关键可维护点

后续维护时，最常改的是下面这几块。

### 6.1 作者人工别名

文件：

- `analyze_2025_esi.py`

变量：

```python
AUTHOR_ALIAS_MANUAL = {
    "wupeipei": "吴佩佩",
    "lirongrong": "李荣荣",
    "chenafei": "陈阿飞",
}
```

用途：

- 某些作者在附表中只写拼音，且无法从当年其他记录自动学到中文名时，可以手工补进这里

### 6.2 特殊题名学科回填

变量：

```python
MANUAL_SUBJECT_OVERRIDE_RAW
```

用途：

- 第1期附表缺学院/学科字段时，或者个别论文后续期次没有补齐学科时，可以在这里手工指定

### 6.3 待核拼音作者白名单

变量：

```python
MANUAL_PENDING_AUTHOR_OVERRIDES_RAW
```

用途：

- 控制哪些“部分中文 + 部分拼音”的论文要在总表和说明文档中显式列出待核作者

### 6.4 文件名模式

函数：

```python
build_sources()
```

用途：

- 如果以后附表命名规则变了，需要修改这里的 glob 模式

## 7. Windows 部署方式

### 7.1 方式一：直接运行 bat

适合：

- 校内电脑已安装 Python
- 先验证原型是否可用

步骤：

1. 安装 Python 3.11 或 3.12
2. 进入项目目录
3. 双击：

```text
windows_tool\run_esi_tool.bat
```

这个脚本会自动：

- 创建 `.venv`
- 安装依赖
- 启动桌面 GUI

### 7.2 方式二：打包成 exe

适合：

- 非技术老师长期使用
- 不希望每台电脑都装 Python

步骤：

1. 在一台已安装 Python 的 Windows 电脑上执行：

```text
windows_tool\build_windows_exe.bat
```

2. 打包完成后，到：

```text
dist\ESI统计工具\
```

3. 把整个目录拷到老师电脑上即可

注意：

- PyInstaller 打包时，建议在 Windows 本机上执行，不建议跨平台打包
- 如果目标机器缺少 Office/字体，docx 的视觉效果可能和开发机略有差异

## 9. 常见问题

### 9.1 提示找不到第1期或第6期附表

原因：

- 文件名与当前 glob 规则不匹配

处理：

- 检查文件名中是否包含 `第1期附表`、`第6期附表`
- 必要时修改 `build_sources()` 中的匹配规则

### 9.2 作者名字还是拼音

原因：

- 当前附表里没有给中文括注
- 也没有在人工别名表中维护

处理：

1. 先查校内可靠公开页面
2. 确认后补到 `AUTHOR_ALIAS_MANUAL`
3. 重新运行工具

### 9.3 正式报告能生成，但格式和开发机略有差异

原因：

- Windows 机器上的 Office、字体或 Word 渲染环境不同

处理：

- 尽量统一安装 Office
- 保持微软雅黑、宋体、仿宋、黑体等常用中文字体齐全

### 9.4 bat 双击后闪退

原因通常有三个：

1. 电脑没装 Python
2. 公司或学校安全策略拦截了脚本执行
3. 网络环境导致依赖安装失败

处理：

- 先在命令行运行 `py --version`
- 再手动执行 `windows_tool\run_esi_tool.bat` 看报错

## 10. 现阶段限制

当前原型仍有这些限制：

1. 年度文案仍按 `2025` 写死
2. 统计口径默认依赖 `2024` 年正式报告做作者种子映射
3. 文件匹配规则仍按当前 ESI 附表命名习惯编写
4. 表14仍是“代表作者表”，不是“全作者展开表”

这意味着它已经适合做 `2025` 年这套流程的一键化，但若要扩成“任意年度通用平台”，还需要再做一轮参数化。

## 11. 下一步建议

如果要继续迭代，我建议按这个顺序：

1. 把 `2025`、`2024模板`、`文件模式` 提到外部配置文件
2. 增加 `表14补充附表`，把代表作者之外的其余本校作者显式列出
3. 在 GUI 中增加“作者待核人工确认”窗口
4. 最后再做正式的 exe 打包与校内分发

这样风险最低，也最符合老师真实使用场景。
